# Admission and Reuse Receipts

This directory documents the fail-closed receipt process. Machine-readable indexes live in `registry/` and begin empty.

- A source-admission receipt governs public rules, datasets, opinions, filings, or other external material.
- A reuse-release receipt governs every cross-repository code, prompt, template, guideline, or workflow port.
- Candidate, unknown, expired, or missing receipts grant no authority.
- Approval of review does not approve copying. Each reuse receipt must identify an exact source commit and exact paths.
- Raw source rows, copyrighted text, client data, credentials, and private lineage evidence do not belong here.

The current empty registries are intentional evidence that this build has admitted no external sources and copied no gated local assets.
